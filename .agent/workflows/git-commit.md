# Git 커밋 규칙

## 커밋 메시지
- **커밋 메시지는 반드시 한글로 작성**해야 합니다.
- 영어로 작성하지 않습니다.

### 예시
✅ 올바른 예시:
```
feat: 예약 취소 기능 추가
fix: 로딩 상태 버그 수정
refactor: 공통 유틸리티 함수 분리
style: 버튼 스타일 개선
```

❌ 잘못된 예시:
```
feat: add reservation cancel feature
fix: loading state bug fix
```

## 커밋 타입
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링
- `style`: 스타일/UI 변경
- `docs`: 문서 수정
- `chore`: 기타 작업
